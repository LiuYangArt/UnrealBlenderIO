# UnrealBlenderIO AGENTS

## 本地化规则

本项目的 Blender 插件界面必须长期保持“跟随 Blender 语言自动切换”的能力。以后新增功能、修改现有功能、补充设置项或提示文案时，必须遵守以下规则。

### 1. 总原则

1. 所有面向用户的可见文本都必须参与本地化，不能再直接在业务代码里硬编码中文或英文句子。
2. 本项目当前只维护两种语言：
   - `zh_HANS`
   - `en_US`
3. 插件显示语言必须跟随 Blender 当前语言切换，不能单独再做一套插件内部语言开关。

### 2. 文案存放规则

1. 中文和英文文案必须分别保存在独立 JSON 文件中。
2. 两份 JSON 使用完全一致的 key，不允许一边多、一边少。
3. 新增文案时，必须同时补齐：
   - 中文词条
   - 英文词条
4. 词条 key 必须稳定、可读、可复用，禁止使用临时命名。

推荐命名方式：

- `panel.*`：面板标题、分组标题
- `prop.*`：属性名、属性说明
- `op.*`：操作器名称、操作器描述
- `report.*`：运行时提示、警告、错误
- `hint.*`：模态操作提示、悬浮说明
- `pref.*`：插件设置页文案

### 3. 必须本地化的内容

以下内容全部视为“用户可见文本”，新增或修改时必须走本地化：

1. `bl_label`
2. `bl_description`
3. `StringProperty` / `BoolProperty` / `EnumProperty` 等属性的 `name` 和 `description`
4. 面板里的 `label(text=...)`
5. 按钮、菜单、操作项名称
6. `self.report(...)` 中的提示、警告、错误
7. `blf.draw(...)` 或其他自绘 UI 中显示的说明文字
8. 设置页中的说明文本

### 4. 禁止事项

1. 禁止在 `UI.py`、`Tools.py`、`UnrealBlenderIO.py`、`Preference.py` 这类 Blender 插件文件中新增硬编码中文提示。
2. 禁止在这些文件中新增硬编码英文 UI 文案，然后指望以后再补翻译。
3. 禁止出现“一个功能只有中文，没有英文”或“只有英文，没有中文”的提交。
4. 禁止中英混杂地拼接一句完整用户提示，除非术语本身必须保留英文，例如 `JSON`、`FBX`、`Unreal Engine`、`Blender`。

### 5. 开发要求

以后新增功能时，开发顺序必须是：

1. 先确定新增了哪些用户可见文案。
2. 先补两份 JSON 词条。
3. 再在代码里通过统一的本地化接口引用这些词条。
4. 最后检查 Blender 中文和英文界面下是否都能正确显示。

### 6. 一致性要求

1. 中英文 JSON 的 key 集必须完全一致。
2. 同一条文案的占位符必须一致，例如中文和英文都使用 `{filename}`，不能一边是 `{file}` 一边是 `{filename}`。
3. 同一术语要保持统一，不要同一个概念在不同文件里反复换说法。

建议统一术语：

1. `Unreal Scene`
2. `Level Asset Collection`
3. `Proxy Pivot`
4. `UBIO Temp Files`

如果中文需要本地化表达，应在两份 JSON 中成对维护，不要临时在代码里改字面量。

### 7. 提交前检查

每次涉及用户可见文本的改动，提交前至少检查下面几项：

1. 是否补齐了 `zh_HANS` 和 `en_US` 两份词条。
2. 是否仍有新的硬编码中文或英文 UI 文案残留在 Python 代码里。
3. 是否会随 Blender 语言切换正确显示。
4. 是否存在漏翻译、错 key、占位符不一致的问题。

### 8. 例外范围

以下内容可以不进入本地化系统：

1. 纯内部调试日志
2. 不面向最终用户的开发注释
3. Unreal 侧脚本里仅供开发排查的内部输出

但只要文本会直接显示在 Blender 插件界面或 Blender 操作反馈里，就必须进入本地化。
